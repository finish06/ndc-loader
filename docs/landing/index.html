Features
Everything you need for drug data
Replace your openFDA dependency with a self-hosted API that's faster, more reliable, and fully under your control.
📦

FDA Data Ingestion

Automated daily download of NDC Directory and Drugs@FDA datasets. Atomic swap ensures consumers never see partial data. Checkpoint retry for reliability.

🔍

NDC Lookup

Any NDC format accepted — hyphenated, unhyphenated, 2-segment, 3-segment. Handles all FDA patterns: 4-4-2, 5-3-2, 5-4-1. Sub-5ms P95 latency.

⚡

Full-Text Search

PostgreSQL-powered search across brand names, generic names, and manufacturers. Prefix matching, relevance ranking, and pagination built in.

🔄

openFDA Compatible

Drop-in replacement for /drug/ndc.json. Same response format, same search syntax. Zero code changes for existing consumers.

API
Simple, powerful endpoints
RESTful API with API key authentication. Explore interactively via the built-in Swagger UI.

Quick Start
Up and running in minutes
rx-dag runs as a Docker Compose stack with PostgreSQL.
1

Clone and configure

Clone the repository and set your API key.

git clone https://github.com/calebdunn/ndc-loader.git
cd ndc-loader
export API_KEYS="your-secret-key"
2

Start the stack

Docker Compose brings up PostgreSQL and ndc-loader together.

docker-compose up -d
3

Trigger a data load

Kick off the initial FDA data ingestion (runs daily automatically after this).

curl -X POST -H "X-API-Key: your-secret-key" \
  http://localhost:8081/api/admin/load
4

Query drug data

Once loaded, start querying the API.

curl -H "X-API-Key: your-secret-key" \
  http://localhost:8081/api/ndc/search?q=aspirin
